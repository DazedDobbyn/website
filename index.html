HTML Roles

HTML Debugging

Debugging is the process of finding and fixing errors in a given code. It's named this way due to early versions of computers often being infested with literal bugs, then needing to be "debugged". Later, when the coding errors became greater than the physical ones, the term slowly had its meaning changed from insect bashing to error fixing.

HTML Attributes

An HTML Attribute refers to the additional information in the start tag of a specific HTML element. For example, you can't use the <a> tag to link to stuff without the href attribute. Likewise, you can't size an image to your liking at least without the width and height attributes.

ARIA Roles

Accessible Rich Internet Applications, or ARIA, is a set of attributes/roles that help the machine recognise and provide certain elements in web pages with accessibility features for the disabled, such as the ability to be screen-read.

HTTP or HTTPS?

HTTP, or Hypertext Transfer Protol, is an application protocol that enables most of the data communication on the web. HTTPS, or HTTP Secure, does exactly the same, but encrypts and verifies the requests and responses being transferred. HTTPS is thus far more secure than HTTP, and should always be used instead.

URL paths

Absolute and Relative URLs

  • Absolute URLs are links that include the protocol, the domain, and the location within the website in the folder system, e.g., "https://www.website.com/xyz.html". Relative URLs only include the location within the folder that comes after the domain, e.g.,"/xyz.html". Because of this, relative URLs can only be used to link to different pages/locations on the same site under the same root domain. They are, however, much easier to code with, and save a lot of time over using absolute URLs to link to local files.

Index.html

  • When accessing a site, the URL that was used typically won't have a specific file to point to. E.g., website.com, as opposed to website.com/gallery.html. As a result the webserver will—by default—try to locate a file named index.html, as this is the most common name used for default pages (home page) on websites. This can be configured by editing the webservers config files.

Images and Graphics

Images

The <img> tag is used—alongside four attributes—to refer to an image on a webpage. The four attributes are width, height, src, and alt. The width and height attributes are pretty straightforward, requiring you to set a size for the image on your page. The src attribute means "source" and is where you enter the location of the image folder in the webserver. The alt attribute gives text to an image, and is the alternative to it for screen-readers, or if the image doesn't load.

Image Formats

Images can come in many different formats, each with their own pros and cons. It's important to choose the right image format that suits your needs, as some formats can offer great quality images, but at the expense of file size, and vice verse. Below is a list of some of the most common image formats...

  • JPG - The most common image format. Known for its 'lossy' compression technique which decreases image quality as the file size drops.
  • PNG - Another very common image format that is 'lossless' meaning it retains image quality regardless of editing. As a result, PNG file sizes tend to be larger than JPGs.
  • GIF - Also uses lossless compression. However, since GIF files are only 8-bit, they can only display up to 256 colours (as opposed to JPGs 24-bit files which can display 16,777,216 colour variations). This makes the GIF format the go-to when creating short animated content, as file sizes stay relatively small due to lower quality.
  • RAW - The first image format that gets processed on your camera when you take a photo, to be stored for reformatting later. This is the highest quality format, as it is literally the photo you took, without any sort of compression. This of course results in very large file sizes, and is only typically used for post-production and editing.
  • WebP - A new lossy and lossless image format created by Google to replace the JPG, PNG and GIF formats. It offers smaller file sizes and better quality, but has yet to be supported by most major browsers.

Bitmap vs Vector Graphics

Bitmap graphics work by telling each pixel on the screen what colour they should be, whereas Vector graphics works by telling each component on the screen where to go. This means that Vector images can be resized at no loss of quality, unlike Bitmap, which requires pixels to "stretch out" to meet the image size, resulting in a blurry look. Bitmap images are, however, much better at producing larger variations of colours, like in most photographs. Vector images are only ideal for where the image has little colour variation going on.

The srcset Attribute

The srcset attribute defines a list of image file URLs, along with specific sizes for each image. When the browser gets resized, the image will change depending on, say, the mid-width attribute. There also needs to be a default image identified with the standard src attribute for browsers that don't support srcset.

Responsive Images

Responsive images is simply a set of techniques used to keep your imaged sized correctly with the browser. This makes for a more pleasant viewing experience, but can be annoying to implement into your page properly.
There is also the ability to change images based on the size of the window. You can use the media attribute along with srcset to define a minimum window size for one or more images to be displayed on the page.

Figure and Figcaption

You can use the <figure> tag to markup an image on your page, and use the <figcaption> tag to define a caption for the image.

Putting It All Together

HTML Page

Every HTML Page is barebones structured the same. They consist of four key elements in order to function, being:

  • !DOCTYPE - Not a tag, but instead a declaration of what type of document the browser should expect, the given being the latest document type.
  • <html> - The first and last element on the page, representing the root tag for all other content in the HTML page (except !DOCTYPE.
  • <head> - A container for metadata, which is found between the <html> and <body> tag. This data isn't displayed, but insteads contains information about the other data in the page, such as the doc title, styling, character sets, etc.
  • <body> - All data in here will be displayed on the website.

Document Head

The Document Head contains all metadata relating to the HTML page, consisting of elements such as:

  • <meta> - This defines metadata about the HTML document, such as the character set, viewport settings, keywords, and the author of the document.
  • <title> - Your webpage title, which will be displayed in the tab indicator in the browser and is indexed by search engines.
  • <link> - Tells the browser where external pages are located to link to, such as CSS Stylesheets.

Structuring Content

Structuring a webpage can be a smooth and easy process when using what's called semantic elements. These elements clearly describe their meaning and the content they should hold. For example, a <div> tag isn't semantic, as it could hold any kind of content. A <nav> tag, or perhaps a <footer> tag—both which give their meaning in their own names—are semantic, as the <nav> tag is purposed for navigational content, and the <footer> tag is purposed for content that goes in the footer.

The Web

When was the web invented, and by who?

The World Wide Web (WWW) was first created in 1989 by British scientist Tim Berners-Lee, for the purpose of automated information-sharing between scientists and institutes.

What are the three main language of the web?

HTML, CSS and Javascript are the three tools any web developer needs in order to do their work.
HTML so you can tell the browser where to put what, CSS so you can tell the browser how it looks, and Javascript so the browser actually understands what you're saying.

How does a local HTML page become public?

By using what's called a CMS, or Content Management System. Wordpress is one example of an open-source CMS, which hosts roughly a quarter of all websites on the entire web.

What is an IDE?

An IDE (Integrated Development Environment) is a software that allows someone to write and test software, as well as facilitating both processes through means such as autofilling elements, syntax highlighting and automatic structuring.
